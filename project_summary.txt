Project Name: Lorrey (Logistics & Transport Management System)

1. PROBLEM
Managing a transport and logistics business involves complex, often disjointed operations. Key challenges include:
- Manual estimation of trip distances and fuel requirements for lorry hire.
- Segregating operational access between different business units (e.g., Office Administrators vs. Petrol Pump Administrators).
- Lack of real-time synchronization for financial and operational records like GST registers, voucher registers, and cement registers.
- Inefficient handling of payment proofs (receipts) across multiple portals.
- Fragmented UI for truck and driver management leading to operational friction.

2. SOLUTION
A unified, real-time web application ("Lorrey") that digitizes and centralizes logistics management. The system provides role-specific workspaces, automates critical calculations (like fuel tracking), provides spreadsheet-like interfaces for live data entry, and securely manages document uploads (multi-proof payment receipts).

3. APPROACH
- Real-time Data Sync: Utilizing WebSockets to ensure that data entry in registers (Cement, Voucher, GST) is instantly updated across all connected clients.
- Modular Architecture: Separating the frontend interfaces based on roles (Office Admin vs. Pump Admin) while maintaining a centralized Backend and AI Worker service.
- Secure Document Flow: Integrating cloud storage for secure handling of multi-file payment proofs and excel exports.
- Automated Workflows: Implementing algorithms to calculate shipping distances dynamically and estimate fuel costs automatically to reduce human error.

5. DETAILED FRAMEWORKS & TECHNOLOGIES
The project is built on a modern, scalable web stack utilizing the following frameworks:

**Frontend Ecosystem**
- Core Framework: React.js (Component-based architecture for dynamic user interfaces).
- UI/Styling: Customized CSS framework enabling a distinct "purple-themed" design language, complete with smart Autocomplete components for data entry.
- Real-Time Integration: Socket.io-client (Enables instant register updates without page reloads).
- File Handling: Client-side file processing for multi-proof images/PDFs prior to upload.

**Backend Ecosystem**
- Runtime environment: Node.js (High-performance asynchronous JavaScript runtime).
- Web Framework: Express.js (Handles robust API routing, middleware, and request validation).
- Real-Time Server: Socket.io (Broadcasting live updates to all connected office/pump admins).
- Excel/Data Processing: ExcelJS (or equivalent library) for generating dynamic, true .xlsx exports directly from database queries.

**Database & Cloud Infrastructure**
- Database: MongoDB (NoSQL document structure is ideal for flexible register schemas and multi-portal records).
- Object Modeling: Mongoose ODM (Enforcing schema validation for hire slips, transaction registers, and multi-file arrays).
- Cloud Storage: AWS S3 (Highly durable object storage used to securely house sensitive payment proofs, GST attachments, and PDF generated invoices).

**Background & AI Services**
- AI Worker Service: A standalone microservice (likely Python or Node-based) acting as a queue processor for computationally heavy AI or background tasks (e.g., smart distance calculation APIs).
